******Conceitos Fundamentais******

Commit: Snapshot("foto") do projeto em um momento específico
Branch: Ramificações do código 
Merge: Combinação de código
Clone: Cópia completa de um repositório
Pull/Push: Sincronização de mudanças entre repositórios

*********************************

Depois que criar uma versão do repositório e eu atualizar ela, é criado outra versão.
Se você apagar uma versão do repositório, ela pode ser criada novamente do zero.

git reset --hard a1b2c3d[Hash da versão] = Utilizado para retornar para a versão anterior, caso a versão nova possua falhas.

1- Main Branch: Branch principal que está sendo executada
2- Develop Branch: Branch de desenvolvimento
3- Feature Branch: Branch de desenvolvimento de cada funcionalidade (tem tempo de produção) [enviada para Develop]
4- Hotflix Branch: Branch de desenvolvimento de cada funcionalidade (Situação crítica do Main Branch) [enviada para Develop e Main]

vim .git/config = Utilizado para ver se a pasta é direto do git

git init = para criar um repositório
git checkout -b "Nome" = utilizada para criar uma nova branch
git branch = usada para listas as branchs criadas
git branch -M main = 
git remote add origin (link) = Para fazer a sincronia da pasta com repositório
git push -u origin main = usado para fazer a cópia da pasta no branch main
git config --global user.name "Nome" = Para logar no git
git config --global user.email "Nome" = Para logar no git

JavaScript é padronizado pela ECMA INternacional como ECMAScript
defer = o download do script é feito de forma assíncrona, mas sua execução ocorre apenas quando todo o processo de renderização estiver concluído (Espera o HTML carregar primeiro).
async = possibilita que o download do script seja feito de modo assíncrono, em paralelo com o HTML (imediatamente em conjuto com o HTML).
